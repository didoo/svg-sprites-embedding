<section id="buttons-svg-notes" class="docs background-subtle">
<header >
<h3 class="docs-title"><i class="fa fa-file-text-o"></i> Inline SVG Notes</h3>
</header>

<div class="docs-content">
<p>
The svg-builder directory in this project, contains
a complete inline SVG build workflow. It requires you to have an understanding of things like <a href="http://gruntjs.com/">Grunt</a>
and a bit of comfort with the command line. You may also want to do a bit of reading to get some background on inline SVG works. Here are some hand-picked posts to get you started:
</p>

<ul class="docs-list">
<li>
<a href="http://css-tricks.com/icon-fonts-vs-svg/">Icon Font vs. SVG</a> – this is the one that convinced me it was time to try using inline SVG in a production system.
</li>
<li>
<a href="http://css-tricks.com/using-svg/">Using SVG</a> – just a great overview on SVG in use.
</li>
<!-- <li>
<a href="http://css-tricks.com/ROBS-POST/">ROBS POST TBD</a>–this article lists all gotchas and workarounds I came up against (this <em>WILL</em> save you time if you're doing this &ldquo;for reals&rdquo;)
</li> -->
<li>
<a href="http://css-tricks.com/svg-use-external-source/">SVG Use External Source</a> – information on caching SVG by using an external defs file.
</li>
<li>
<a href="http://css-tricks.com/swapping-svg-icons/">Swapping Icons</a> – some very practical tips on how to, well, swap icons.
</li>
<li>
<a href="http://css-tricks.com/tag/svg/">All CSS-Tricks articles tagged "svg"</a> – since Chris has become a bit of curator on the topic, this is useful too!
</li>
</ul>
<p>Ok…ok, the tl;dr is to hit the terminal and type:
<pre>git clone -b svg-inline-experiments --single-branch https://github.com/unicorn-ui/Buttons.git Buttons && cd Buttons && npm install && pushd svg-builder && npm install && grunt && popd && grunt dev</pre> And then go ahead and have a look at the Gruntfile.js <i class="fa fa-smile-o fa-2x"></i></p>
</div>
</section>
